# 要求定義書：自前Instagramウィジェット（sukusuku-instagram-widget）

## 1. 目的
- 外部サービス（Elfsight等）の広告表示や利用料金、表示回数制限を回避する。
- 完全に無料で、最新のInstagram投稿4件をWebサイト上に表示し続ける。
- 運用の手間を最小限にし、トークンの更新を自動化する。

## 2. ターゲット機能
### 2.1 表示機能
- **プロフィール情報**:
  - アカウントアイコン（プロフィール写真）。
  - アカウント名（ユーザーネーム）。
  - 統計情報（投稿数、フォロワー数、フォロー数）。
  - 「フォローする」ボタン（Instagram公式ページへ誘導）。
- **投稿リスト**: 最新のInstagram投稿（画像または動画のサムネイル）を **最大4件** 表示する。
- 各投稿をクリックすると、Instagramの該当投稿ページへ遷移する。
- 投稿のキャプション（冒頭部分）をホバー時または下部に表示する（デザインによる）。

### 2.2 デザイン要件
- **Elfsightウィジェットのデザインを完全踏襲しつつ、一部カスタマイズする。**
  - **プロフィールエリア**: 上部にアイコン、名前、統計、ボタンをバランスよく配置。
  - **投稿エリア**: 2×2のグリッド配置。
  - **形状**: 投稿画像に **角丸を適用せず、純粋な正方形** で表示する。
  - サイト全体のトーン＆マナーに合わせ、違和感のないフォントと配色を適用する。

### 2.3 運用・保守機能
- **トークン自動更新**: Meta APIの長期トークン（60日間有効）を、GitHub Actionsを用いて30日ごとに自動リフレッシュする。
- **データ自動更新**: GitHub ActionsでInstagram APIから最新情報を取得し、JSONファイルとしてリポジトリ内に保存・プッシュ（毎日19:00に実行）。これにより、フロントエンドからのAPI直接呼び出し（トークン露出）を回避する。

## 3. 非機能要件
- **コスト**: 完全無料（Metaデベロッパーアカウント、GitHub Actions無料枠の範囲内）。
- **パフォーマンス**: 外部ウィジェット読み込み待ちを排除し、サイト自体の読み込み速度を向上させる。
- **セキュリティ**: APIトークンをフロントエンド（ブラウザ上で閲覧可能なコード）に露出させない。

## 4. 制限事項
- InstagramのAPI仕様変更に伴うメンテナンスが発生する可能性がある。
- 動画投稿の場合、プレビュー画像（サムネイル）の取得に制限がある場合があるが、基本は最新のメディアURLに準拠する。
